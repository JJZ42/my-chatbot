### 2.2 知识库函数

| | V1.0 | V2.0 |
| --- | --- | --- |
| 函数名 | `create_knowledge_base(texts)` | `compute_tfidf(texts_tuple)` |
| 缓存 | 无 | `@st.cache_resource` |
| 参数 | `texts: list` | `texts_tuple: tuple`（hashable） |
| 分词器 | `TfidfVectorizer()` | `TfidfVectorizer(analyzer='char_wb', ngram_range=(1,3))` |

**关键差异 — 中文分词：**

```python
# V1.0：默认分词 → 中文挤成1个token → 检索全0分
vectorizer = TfidfVectorizer()

# V2.0：逐字分词 + 1~3字ngram → 正确匹配中文
vectorizer = TfidfVectorizer(analyzer='char_wb', ngram_range=(1, 3))
```

**效果对比（查询 "list是什么"，知识库有12条Python知识）：**

| | V1.0 | V2.0 |
| --- | --- | --- |
| No.1 | 异常处理 (score: **0.0000**) | for循环 (score: **0.3093**) |
| No.2 | 文件读取 (score: **0.0000**) | 列表list定义 (score: **0.2606**) |
| No.3 | pip安装 (score: **0.0000**) | pip安装 (score: **0.1863**) |
| 结论 | 🔴 完全随机 | 🟢 正确匹配 |

### 2.4 主对话区

```python
# V1.0：每次只发当前问题（无对话记忆）
messages=[{"role": "user", "content": prompt}]

# V2.0：发送完整对话历史 + 当前RAG增强问题
chat_messages = [
    {"role": "user", "content": "指针是什么"},       # 历史Q1
    {"role": "assistant", "content": "指针是..."},   # 历史A1
    {"role": "user", "content": "RAG上下文 + 它和引用的区别"}  # 当前Q2
]
```

## 五、已知限制

| 限制 | 说明 |
| --- | --- |
| 刷新持久化范围 | `st.session_state` 级别 — 服务器重启后丢失 |
| 中文分词精度 | `char_wb` 是字符级，不如 jieba 等专业分词器 |
| 知识库大小 | 内存存储，超大知识库（>1000条）可能卡顿 |
| 对话记忆开销 | 每轮都发全部历史，长对话 token 消耗递增 |
